HCI-HW3

구글포토

A. Requirements

1. Describe the overall objective of the application (one paragraph)

스마트폰의 등장으로 이미지를 공유가 제일 활발해졌다. 이미지 공유하기 쉬어지면서 현재 보고있는 이미지를 저장하거나 현재 화면을 캡쳐할 때 손쉬운 몇 번의 터치로 관리를 제공해 줄 수 있는 기능이 필요해졌다. 이 앱은 이러한 기능들을 제공한다.

2. List major functional requirements (3 ~ 4)

  • 다운받으면서 태그하기 : 기존에는 웹에서 이미지를 다운로드하면 스토리지로 바로 저장이 된다. 이 앱은 웹에서 이미지를 다운 받을 때 원하는 태그를 선택하거나 태그를 추가해서 저장할 수 있다.
  • 캡쳐하고 태그하기 : 이미지를 캡쳐하면 바로 싀토리지에 저장되지 않고 태그를 선택하거나 추가해서 저장할 수 있다.
  • 태그로 사진들 분류해서 보기 : 태그로 저장된 사진들을 분류해서 볼 수 있다.
  • 기존 사진들 태그하기 : 기존에 있던 사진들에 태그를 추가해서 분류할 수 있다.

3. List major UI requirements

  1. Functional
    • 터치엔 입력 : 저장하는 화면에서 쉽게 태그를 눌러서 선택할 수 있어야 한다.
    • 쉽게 태그를 입력할 수 있어야 한다.
    • 스와이프 : 태그들이 스와이프 할 수 있도록 명시적으로 보여야 한다.
    • 진행아이콘 : 추천태그는 처리속도가 느릴 수 있으니깐 진행아이콘으로 태그가 나올 것이란 것을 알려주어야 한다.
  2. Non-Functional
    • 테마선택 : 사진을 주로 사용하는 층은 꾸미는 것에 관심이 많을 수 있으므로 테마를 선택할 수 있어야 한다.
    • 태그 사이징 : 태그의 사이즈를 조절 할 수 있어야 한다.

4. List other considerations

  • 스마트폰으로 이미지를 자주 저장하는 10대에서 30대 후반까지
  • 스마트폰 환경에서 작동하는 안드로이드/IOS 앱
  • 스마트폰이 익숙한 사람들
  • 이미지를 많이 공유하는 사람들

B. User analysis

1. Conduct user interviews (2-3 people) from your target user group. Recode a short video of the interview (hide the interviewee)

2. List the major direct user requirements? (3 ~ 4)

  1. 자동으로 정렬 ai
  2. 위치도 자동으로 추가되면 좋겠다
  3. 옵션으로 자동추가가 되었으면 좋겠다.
  4. 구글 드라이브같은 클라우드와 연동되었으면 좋겠다.

3. List the major cognitive/ergonomic requirements? (more than 2)

  1. 최근태그들의 크기로 중요도를 알려줬으면 좋겠다.

  2. 글씨키가나 컬러가 변경되었으면 좋겠다.

    4. Construct 2-3 usage scenarios. Use Powerpoint or any drawing software and annotate important issues.

  3. 가게에서 음식 사진을 찍고 바로 태그를 달아 저장할 때

  4. 인터넷에 짤줍할 때

    1. 웹서핑 중 원하는 이미지를 발견한다.
    2. 이미지 공유를 누른다
    3. 앱을 클릭한다.
    4. 태그를 입력하여 저장한다.

5. Construct a rough interaction model (e.g. like a GOMS model) for 2-3 major tasks.

1번이나 2번 시나리오나 GOMS 모델은 다르지 않으므로 한번만 모델링해본다. 또한 세번의 태그를 한다고 가정하고 태그를 누를 때 고민시간은 넣지 않는다. 이런 분류는 아무리 인터페이스가 훌룡할 지라도 태그를 고민하는 시간이 더 중요하기 때문에 GOMS모델링은 최소시간만 계속될 수 있다.
