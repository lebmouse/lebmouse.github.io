HCI4-2

A. Requirements

1. he overall objective of the application

대청소를 하기보단 제자리에 물건을 두는 편이 집을 관리하기 편하듯이 사진정리도 마찬가지다. 스마트폰 덕에 직접 사진을 찍거나 인터넷의 이미지를 언제 어디서든 공유할 수 있다. 이미지를 나중에 공유하기 위해서 단순히 이미지를 마구 저장하기도 하는데 그 저장하는 과정에서 손쉽게 분류할 수 있으면 나중에 찾아보기 쉽고 나중에 정리할 필요가 줄어들 것이다.

2.major functional requirements (3 ~ 4)

  • 다운받으면서 태그하기 : 기존에는 웹에서 이미지를 다운로드하면 스토리지로 바로 저장이 된다. 이 앱은 웹에서 이미지를 다운 받을 때 원하는 태그를 선택하거나 태그를 추가해서 저장할 수 있다.
  • 캡쳐하고 태그하기 : 이미지를 캡쳐하면 바로 스토리지에 저장되지 않고 태그를 선택하거나 추가해서 저장할 수 있다.
  • 태그로 사진들 분류해서 보기 : 태그로 저장된 사진들을 분류해서 볼 수 있다.
  • 기존 사진들 태그하기 : 기존에 있던 사진들에 태그를 추가해서 분류할 수 있다.

3. List major UI requirements

  1. Functional
    • 태그 버튼 : 색의 변화로 on/off
    • 상태바 겸 태그 인풋창 : 텍스트로 입력 가능해야하고 현재 선택된 태그상태가 나타나야 함.
    • 반투명 프레임 : 어떤 이미지를 저장하는지 알 수 있게 반투명으로 된 프레임을 가져야 함
    • 저장버튼
    • 설정버튼
    • 스와이프 가능한 버튼 그룹 박스
  2. Non-Functional
    • 진행아이콘 : 추천태그는 처리속도가 느릴 수 있으니깐 진행아이콘으로 태그가 나올 것이란 것을 알려주어야 한다.
    • 테마선택 : 사진을 주로 사용하는 층은 꾸미는 것에 관심이 많을 수 있으므로 테마를 선택할 수 있어야 한다.
    • 태그 사이징 : 태그의 사이즈를 조절 할 수 있어야 한다.
    • 버튼의 색은 대조적이어야 한다.

4. List other considerations

  • 스마트폰으로 이미지를 자주 저장하는 10대에서 30대 후반까지
  • 스마트폰 환경에서 작동하는 안드로이드/IOS 앱
  • 스마트폰이 익숙한 사람들
  • 이미지를 많이 공유하는 사람들
